Singly Linked List — one-directional pointers
Speed: Fast
Ready. Switch between Singly and Doubly modes above.

Singly linked: each node holds data and one pointer to the next node. No pointer back. Traversal is one-directional. Add/delete at head is O(1); all other position operations need O(n) traversal first.

Pseudocode — highlighted lines follow the animation
NODE: data, next pointer SINGLY LINKED LIST: head pointer PROCEDURE addFirst(val): new_node ← create node(val) new_node.next ← head head ← new_node END PROCEDURE PROCEDURE addAt(index, val): IF index = 0 THEN addFirst(val) cur ← head, step ← 0 WHILE step < index - 1: cur ← cur.next step ← step + 1 new_node.next ← cur.next cur.next ← new_node END PROCEDURE PROCEDURE deleteAt(index): IF index = 0 THEN head ← head.next cur ← head, step ← 0 WHILE step < index - 1: cur ← cur.next cur.next ← cur.next.next END PROCEDURE PROCEDURE reverse(): prev ← null, cur ← head WHILE cur ≠ null: next ← cur.next cur.next ← prev prev ← cur cur ← next head ← prev END PROCEDURE
NODE: data, next, prev pointer DOUBLY LINKED LIST: head, tail PROCEDURE addFirst(val): new_node ← create node(val) new_node.next ← head IF head ≠ null THEN head.prev ← new_node head ← new_node END PROCEDURE PROCEDURE addAt(index, val): cur ← head, step ← 0 WHILE step < index - 1: cur ← cur.next new_node.next ← cur.next new_node.prev ← cur IF cur.next ≠ null THEN cur.next.prev ← new_node cur.next ← new_node END PROCEDURE PROCEDURE deleteAt(index): cur ← node at index IF cur.prev ≠ null: cur.prev.next ← cur.next IF cur.next ≠ null: cur.next.prev ← cur.prev END PROCEDURE
