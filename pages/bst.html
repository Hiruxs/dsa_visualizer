Binary Search Tree

Left subtree < root < right subtree at every node.

Search O(log n)Insert O(log n)Worst O(n)
BST — ordered binary tree
Speed: Fast
Ready.

BST property: every node's left subtree contains only smaller values; right subtree contains only larger values. This halves the search space at each step — O(log n) for balanced trees.

Pseudocode — highlighted lines follow the animation
BST property: left < node < right PROCEDURE insert(node, val): IF node is null THEN RETURN new node(val) IF val < node.val THEN node.left ← insert(node.left, val) ELSE IF val > node.val THEN node.right ← insert(node.right, val) RETURN node END PROCEDURE PROCEDURE search(node, val): cur ← root WHILE cur is not null: IF val = cur.val THEN RETURN found IF val < cur.val THEN cur ← cur.left ELSE cur ← cur.right RETURN not found END PROCEDURE
