Quick Sort

Pick a pivot, partition smaller elements left / larger right, recurse on each side.

Average O(n log n)Worst O(n²)Space O(log n)
Quick Sort — partition around a pivot
Speed: Fast
Ready — use Step to go one step at a time, or Auto to run fully.

The partition step is the heart of Quick Sort — it places one element (the pivot) in its final position in O(n). Recursively doing this gives O(n log n) on average. Pivot choice greatly affects performance.

Pseudocode — highlighted lines follow the animation
QUICKSORT(A, p, r) 1 if p < r 2 q = PARTITION(A, p, r) 3 QUICKSORT(A, p, q - 1) 4 QUICKSORT(A, q + 1, r) PARTITION(A, p, r) 1 pivot = A[r] // last element as pivot 2 i = p - 1 // boundary index 3 for j = p to r - 1 4 if A[j] <= pivot 5 i = i + 1 6 exchange A[i] with A[j] 7 exchange A[i+1] with A[r] // place pivot 8 return i + 1
