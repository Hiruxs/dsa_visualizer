Heap

A complete binary tree satisfying the heap ordering property (max heap).

Insert O(log n)Extract O(log n)Peek O(1)
Heap — tree and array stay in sync
Speed: Fast

Parent is always greater than or equal to both children. Insert adds to the end then bubbles up. Extract swaps root with last element, removes it, then sifts down. Both O(log n) because the heap height is O(log n).

Pseudocode — highlighted lines follow the animation
MAX-HEAP: parent >= both children arr[0..n-1], heap_size = 0 // Insert — bubble up O(log n) HEAP-INSERT(A, val) 1 A[heap_size] = val 2 heap_size = heap_size + 1 3 i = heap_size - 1 4 while i > 0 5 p = (i - 1) / 2 6 if A[p] < A[i] 7 exchange A[i] with A[p] 8 i = p 9 else break // Extract Max — sift down O(log n) HEAP-EXTRACT-MAX(A) 1 top = A[0] 2 A[0] = A[heap_size - 1] 3 heap_size = heap_size - 1 4 MAX-HEAPIFY(A, heap_size, 0) 5 return top
