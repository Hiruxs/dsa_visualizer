Queue

A First In, First Out (FIFO) data structure — like people waiting in a line.

Enqueue O(1)Dequeue O(1)
Visualization
Dequeue ←
← Enqueue
FRONT REAR
Queue has 4 elements. FRONT=10, REAR=40.

Real-world uses: Print job scheduling, CPU process scheduling, breadth-first search (BFS) in graphs, and customer service systems. The first person to join is the first to be served.

Code
class Queue {
  constructor() {
    this.items = [];
    this.front = 0;
  }

  enqueue(val) {
    this.items.push(val);
    // Add to rear — O(1)
  }

  dequeue() {
    if (this.isEmpty()) return null;
    return this.items[this.front++];
    // Remove from front — O(1)
  }

  peek() {
    return this.items[this.front];
  }

  isEmpty() {
    return this.front >=
      this.items.length;
  }
}
FIFO Principle
→
Enqueue: New elements join from the REAR
←
