Queue

First In, First Out — enqueue at rear, dequeue from front.

Enqueue O(1)Dequeue O(1)Peek O(1)
Queue — Linear FIFO structure
Speed: Fast
Ready.

Think of a queue at a ticket counter — the first person in is the first served. Used in BFS, task scheduling, and network packet management.

Pseudocode — highlighted lines follow the animation
QUEUE with array, front=0, rear=-1 PROCEDURE enqueue(val): IF queue is full THEN raise overflow error rear ← rear + 1 data[rear] ← val size ← size + 1 END PROCEDURE PROCEDURE dequeue(): IF size = 0 THEN raise underflow error val ← data[front] front ← front + 1 size ← size - 1 RETURN val END PROCEDURE FUNCTION peekFront(): RETURN data[front] END FUNCTION
