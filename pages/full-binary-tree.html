Full Binary Tree

Every node has exactly 0 or 2 children — never exactly one.

Validate O(n)L leaves → 2L-1 nodes
Full Binary Tree — add pairs to keep it full

Note: A Full Binary Tree defines a structural property only — it does not enforce any ordering of values. A Full Binary Tree is not necessarily a Binary Search Tree. The values here are chosen freely; what matters is that every internal node has exactly two children. Used in expression trees where every operator needs exactly two operands.

Pseudocode — highlighted lines follow the animation
FULL BINARY TREE: every node has 0 or 2 children FUNCTION isFull(node): IF node is null THEN RETURN true // Leaf node is valid IF node.left is null AND node.right is null THEN RETURN true // Both children exist — recurse IF node.left ≠ null AND node.right ≠ null THEN RETURN isFull(left) AND isFull(right) // Exactly one child — NOT full! RETURN false END FUNCTION
