Array

Contiguous block of memory — O(1) access, O(n) insert/delete.

Array — contiguous memory blocks
Speed: Fast
Ready — use Step to go one step at a time, or Auto to run fully.

Each element is stored at a fixed offset from the base address, so arr[i] is always one multiplication away — O(1). Inserting or deleting in the middle requires shifting all subsequent elements.

Pseudocode — highlighted lines follow the animation
ARRAY(A, n) // fixed-size, n elements // Access — O(1) direct address GET(A, i) 1 return A[i] // Insert at index — O(n) shift right INSERT-AT(A, idx, val) 1 for i = size downto idx + 1 2 A[i] = A[i-1] // shift right 3 A[idx] = val 4 size = size + 1 // Delete at index — O(n) shift left DELETE-AT(A, idx) 1 for i = idx to size - 2 2 A[i] = A[i+1] // shift left 3 size = size - 1
