Binary Tree — level-order insertion (BFS)
Speed: Fast
Ready.

A plain binary tree has no ordering rule — values are inserted in level-order (top to bottom, left to right) using BFS. This is different from a BST. Searching requires checking every node — O(n).

Pseudocode — highlighted lines follow the animation
BINARY TREE: head node, no order rule PROCEDURE insert(val): new_node ← create node(val) IF root is null THEN root ← new_node queue ← [root] WHILE queue not empty: cur ← dequeue() IF cur.left is null THEN cur.left ← new_node; RETURN ELSE enqueue(cur.left) IF cur.right is null THEN cur.right ← new_node; RETURN ELSE enqueue(cur.right) END PROCEDURE PROCEDURE search(target): queue ← [root] WHILE queue not empty: cur ← dequeue() IF cur.val = target THEN RETURN found IF cur.left ≠ null: enqueue(cur.left) IF cur.right ≠ null: enqueue(cur.right) RETURN not found END PROCEDURE
