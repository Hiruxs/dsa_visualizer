Max Heap

Parent ≥ both children. Root always holds the maximum value.

Insert O(log n)Extract O(log n)Peek O(1)
Max Heap — tree and array stay in sync
Speed: Fast
Ready — use Step to go one step at a time, or Auto to run fully.

Parent is always greater than or equal to both children. Insert adds to the end then bubbles up. Extract swaps root with last element, removes it, then sifts down. Both O(log n) because the heap height is O(log n).

Pseudocode — highlighted lines follow the animation
MAX-HEAP: A[parent] >= A[children] arr[0..n-1], heap_size = 0 // Insert — bubble up O(log n) MAX-HEAP-INSERT(A, val) 1 A[heap_size] = val 2 heap_size = heap_size + 1 3 i = heap_size - 1 4 while i > 0 5 p = (i - 1) / 2 6 if A[p] < A[i] 7 exchange A[i] with A[p] 8 i = p 9 else break // Extract Max — sift down O(log n) EXTRACT-MAX(A) 1 top = A[0] 2 A[0] = A[heap_size - 1] 3 heap_size = heap_size - 1 4 i = 0 5 while 2i+1 < heap_size 6 j = larger child of i 7 if A[i] >= A[j] then break 8 exchange A[i] with A[j] 9 i = j 10 return top
