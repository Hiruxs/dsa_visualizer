Circular Queue

Last slot wraps back to index 0 — freed slots are reused efficiently.

When REAR reaches the end, the next enqueue wraps to index 0 using (rear+1) % size. Full when (rear+1)%size == front. Empty when both are -1.

Pseudocode — highlighted lines follow the animation
CIRCULAR QUEUE: size=8, front=-1, rear=-1 PROCEDURE enqueue(val): IF (rear+1) % size = front THEN raise overflow (queue full) IF front = -1 THEN front ← 0 rear ← (rear + 1) % size arr[rear] ← val END PROCEDURE PROCEDURE dequeue(): IF front = -1 THEN raise underflow (queue empty) val ← arr[front] IF front = rear THEN front ← rear ← -1 // now empty ELSE front ← (front + 1) % size RETURN val END PROCEDURE FUNCTION peekFront(): IF front = -1 THEN raise error RETURN arr[front] END FUNCTION
